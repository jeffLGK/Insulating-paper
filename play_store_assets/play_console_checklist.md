## 第 6 關：發布流程（關鍵）

### Step 6.1 先走「內部測試」
左側 **「測試 → 內部測試」** → 「建立新版本」

1. 上傳 `app-release.aab`（64 MB）
2. **版本名稱**：`1.0.1`
3. **版本資訊**（給測試者看的）：
   ```
   首次發布版本。
   主要功能：
   - VSCC 認證隔熱紙查詢
   - OCR 標籤辨識
   - 產品比對與收藏
   ```
4. 加測試者電子郵件清單（至少加你自己的 `sch666@gmail.com`）
5. 儲存 → 檢查 → 發布到內部測試

> 通常 **10–60 分鐘**後生效，點測試連結到手機安裝

### Step 6.2 內部測試通過 → 升級「封閉測試」
- 需要 **12 位測試者連續測 14 天**才能進正式版
- 測試者招募可以請朋友加 email，或用公開連結
- 14 天後才能推「正式版」

### Step 6.3 正式版（14 天後）
左側 **「版本 → 正式版」** → 建立版本
- 可以用「從其他版本升級」把內部/封閉測試的 .aab 直接拉進來
- 填寫版本資訊 → 送審
- **審查約 1–7 天**（新帳號首次送審通常較久）

## 常見退件原因 & 避開方法

| 退件原因 | 避開方法 |
|---|---|
| 應用程式名稱使用第三方商標 | ✅ 已加「(非官方)」 |
| 隱私權政策連結失效 | ✅ GitHub Pages 已上線 |
| 資料安全表單與實際行為不符 | ✅ 無蒐集、HTTPS 傳輸已如實填寫 |
| APP 無法啟動／閃退 | 先用內部測試充分測過 |
| 截圖含第三方品牌 LOGO | 可能被退，若截圖有知名品牌隔熱紙型號，最好模糊處理或換通用型 |
| 假冒官方 | ✅ 完整描述已聲明「非 VSCC 官方」 |

## 送出後會收到的通知

- 📧 `noreply@google.com` 會寄「送審中 → 已核准 / 已駁回」
- 駁回會附原因，修改後可重送
- 核准後約 **2–4 小時**全球 Play Store 都能搜到

---

## 之後的更新流程

每次改版發布：
1. `pubspec.yaml` → `version: 1.0.2+3`（+後數字 = versionCode，每次必須遞增）
2. `flutter build appbundle --release`
3. Play Console → 建立新版本 → 上傳 .aab → 填版本資訊 → 送審
